= アカウント一覧
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
使用できます `ListAccounts`オプションのページング サポートを使用して、ストレージ テナント アカウントの全リストを取得します。要素アカウントによりボリュームへのアクセスが可能になります。



== パラメータ

このメソッドには次の入力パラメータがあります。

|===
| Name | 説明 | タイプ | デフォルト値 | 必須 


 a| 
ストレージコンテナを含める
 a| 
仮想ボリューム ストレージ コンテナーは、デフォルトで応答に含まれます。ストレージ コンテナーを除外するには、false に設定します。
 a| 
ブーリアン
 a| 
true
 a| 
いいえ



 a| 
開始アカウントID
 a| 
返される開始アカウントID。このアカウント ID を持つアカウントが存在しない場合は、アカウント ID 順の次のアカウントがリストの先頭として使用されます。リストをページングするには、前の応答の最後のアカウントの accountID + 1 を渡します。
 a| 
integer
 a| 
なし
 a| 
いいえ



 a| 
limit
 a| 
返されるアカウント オブジェクトの最大数。
 a| 
integer
 a| 
なし
 a| 
いいえ

|===


== 戻り値

このメソッドの戻り値は次のとおりです。

|===


| Name | 説明 | タイプ 


 a| 
アカウント
 a| 
アカウントのリスト。
 a| 
xref:reference_element_api_account.adoc[アカウント]配列

|===


== リクエスト例

このメソッドのリクエストは次の例のようになります。

[listing]
----
{
   "method": "ListAccounts",
   "params": {
     "startAccountID"  : 0,
     "limit" : 1000
   },
   "id" : 1
}
----
