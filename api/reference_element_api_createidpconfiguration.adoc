= IdpConfigurationの作成
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
使用することができます `CreateIpdConfiguration`クラスターのサードパーティ ID プロバイダー (IdP) を使用して認証のための潜在的な信頼関係を作成する方法。 IdP 通信には SAML サービス プロバイダー証明書が必要です。この証明書は必要に応じて生成され、この API 呼び出しによって返されます。



== パラメータ

このメソッドには次の入力パラメータがあります。

|===
| Name | 説明 | タイプ | デフォルト値 | 必須 


 a| 
idpメタデータ
 a| 
保存する IdP メタデータ。
 a| 
string
 a| 
なし
 a| 
はい



 a| 
idp名
 a| 
SAML 2.0 シングル サインオンの IdP プロバイダーを識別するために使用される名前。
 a| 
string
 a| 
なし
 a| 
はい

|===


== 戻り値

このメソッドの戻り値は次のとおりです。

|===


| Name | 説明 | タイプ 


 a| 
idpConfigInfo
 a| 
サードパーティの ID プロバイダー (IdP) 構成に関する情報。
 a| 
link:reference_element_api_idpconfiginfo.html["idpConfigInfo"]

|===


== リクエスト例

このメソッドのリクエストは次の例のようになります。

[listing]
----
{
    "method": "CreateIdpConfiguration",
    "params": {
        "idpMetadata": "<?xml version=\"1.0\" encoding=\"UTF-8\"?>
             <EntityDescriptor
               xmlns=\"urn:oasis:names:tc:SAML:2.0:metadata\"
               xmlns:ds=\"http://www.w3.org/2000/09/xmldsig#\"
               xmlns:shibmd=\"urn:mace:shibboleth:metadata:1.0\"
               xmlns:xml=\"http://www.w3.org/XML/1998/namespace\"
               ...</Organization>
             </EntityDescriptor>",
        "idpName": "https://provider.name.url.com"
    },
}
----
