= テストドライブ
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
使用することができます `TestDrives`ノード上のすべてのドライブに対してハードウェア検証を実行する方法。この方法では、ドライブ上のハードウェア障害を検出し、検証テストの結果にそれを報告します。



== パラメータ

使用できるのは `TestDrives`クラスター内で「アクティブ」ではないノード上のメソッド。


NOTE: このテストには約 10 分かかります。

このメソッドには次の入力パラメータがあります。

|===
| Name | 説明 | タイプ | デフォルト値 | 必須 


 a| 
力
 a| 
ノード上のドライブをテストするには true に設定します。
 a| 
ブーリアン
 a| 
なし
 a| 
はい



 a| 
minutes
 a| 
テストを実行する分数を指定します。
 a| 
integer
 a| 
10
 a| 
いいえ

|===


== 戻り値

このメソッドの戻り値は次のとおりです。

|===
| Name | 説明 | タイプ 


 a| 
詳細
 a| 
テスト操作の成功または失敗に関する情報。
 a| 
JSONオブジェクト

|===


== リクエスト例

このメソッドのリクエストは次の例のようになります。

[listing]
----
{
   "method": "TestDrives",
   "params": {
              "force": true,
              "minutes" : 10
             },
   "id" : 1
}
----


== 応答例

このメソッドは、ノード内の各ドライブのテスト結果を含むテーブルを返します。
