= 故障
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
障害オブジェクトには、クラスターで検出された障害に関する情報が含まれます。その `ListClusterFaults`メソッドはクラスター障害情報を返します。



== オブジェクトメンバー

このオブジェクトには次のメンバーが含まれます。

|===
| Name | 説明 | タイプ 


 a| 
ブロックアップグレード
 a| 
障害によりアップグレードがブロックされます。有効な値は次のとおりです。

* true: 障害によりアップグレードがブロックされます。
* false: 障害によってアップグレードがブロックされることはありません。

 a| 
ブーリアン



 a| 
クラスター障害ID
 a| 
各クラスター障害に関連付けられた一意の ID。
 a| 
integer



 a| 
コード
 a| 
検出された特定の障害の障害コード。詳細については、「クラスター障害コード」を参照してください。
 a| 
string



 a| 
data
 a| 
障害固有の追加情報。
 a| 
JSONオブジェクト



 a| 
date
 a| 
UTC+0 形式の現在の時刻。
 a| 
ISO 8601文字列



 a| 
詳細
 a| 
追加の詳細を含む障害の説明。
 a| 
string



 a| 
ドライブID
 a| 
ドライブ ID リストの最初のドライブ ID。  driveIDs リストが空の場合 (つまり、ドライブに関連する障害が返されなかった)、この値は 0 になります。
 a| 
integer



 a| 
ドライブID
 a| 
この障害が参照するドライブの driveID 値のリスト。ドライブに関連する障害のために含まれています。ない場合は空の配列になります。
 a| 
整数配列



 a| 
ノードハードウェア障害ID
 a| 
クラスター上のハードウェア障害に割り当てられた識別子。
 a| 
integer



 a| 
nodeID
 a| 
この障害が参照するノードのノード ID。ノードおよびドライブの障害の場合に含まれ、それ以外の場合は 0 に設定されます。
 a| 
integer



 a| 
resolved
 a| 
障害の解決ステータス。有効な値は次のとおりです。

* true: 障害は検出されなくなりました。
* false: 障害はまだ存在しています。

 a| 
ブーリアン



 a| 
解決日
 a| 
障害が解決された日時。
 a| 
ISO 8601文字列



 a| 
サービスID
 a| 
障害に関連付けられたサービス。障害がサービスに関連付けられていない場合、この値は「0」(ゼロ) になります。
 a| 
integer



 a| 
severity
 a| 
障害の重大度。有効な値は次のとおりです。

* 警告: 軽微な問題です。クラスターは機能しており、この重大度レベルではアップグレードが許可されます。
* エラー: 通常はサービスに影響を与えない障害 (パフォーマンスの低下や HA の損失の可能性を除く)。一部の機能が無効になっている可能性があります。
* 重大: サービスに影響を与える重大な障害。システムは API リクエストまたはクライアント I/O を処理できず、データ損失の危険があります。
* ベストプラクティス: 最適ではないシステム構成によって引き起こされる障害。

 a| 
string



 a| 
type
 a| 
障害の種類。有効な値は次のとおりです。

* ノード: ノード全体に影響する障害。
* ドライブ: 個々のドライブに影響する障害。
* クラスター: クラスター全体に影響する障害。
* サービス: クラスター上のサービスに影響を与える障害。
* ボリューム: 個々のボリュームに影響する障害。

 a| 
string

|===
