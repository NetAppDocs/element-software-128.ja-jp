= SolidFireストレージ システム
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
SolidFireオールフラッシュ ストレージ システムは、個別のハードウェア コンポーネント (ドライブとノード) で構成され、各ノードで独立して実行されるNetApp Elementソフトウェアを通じて単一のストレージ リソース プールに結合されます。この統合クラスターは、外部クライアントが使用するための単一のストレージ システムとして提供され、Element ソフトウェア UI、API、およびその他の管理ツールを通じて単一のエンティティとして管理されます。

NetApp Elementソフトウェア ユーザー インターフェイスを使用すると、 SolidFireクラスタのストレージ容量とパフォーマンスを設定および監視し、マルチテナント インフラストラクチャ全体のストレージ アクティビティを管理できます。

image::../media/solidfire_concepts_architecture_image.gif[Element Software のアーキテクチャ図]

SolidFireオールフラッシュ ストレージ システムには、次のコンポーネントが含まれています。

* ノード: クラスターのストレージ リソースを提供する物理ハードウェア。ノードには次の2つのタイプがあります。
+
** ストレージ ノード: ドライブのコレクションを含むサーバー。
** ファイバー チャネル (FC) ノード: ファイバー チャネル スイッチを介して FC クライアントを接続するために使用されます。


* クラスター: 少なくとも 4 つのノードで構成されるSolidFireストレージ システムのハブ。
* 管理ノード: 監視やテレメトリなどのシステム サービスのアップグレードと提供、クラスター アセットと設定の管理、システム テストとユーティリティの実行、トラブルシューティングのためのNetAppサポート アクセスの提供が可能になります。管理ノード (mNode) は、Element ソフトウェア ベースのストレージ クラスターと連携して実行される仮想マシンです。
* Active IQ: クラスター全体のデータの継続的に更新された履歴ビューを提供する Web ベースのツール。特定のイベント、しきい値、またはメトリックに対してアラートを設定できます。  Active IQすると、システムのパフォーマンスと容量を監視し、クラスターの健全性に関する情報を常に把握できます。
* ドライブはストレージ ノードで使用され、クラスターのデータを保存します。ストレージ ノードには 2 種類のドライブが含まれます。
+
** ボリューム メタデータ ドライブには、クラスター内のボリュームやその他のオブジェクトを定義する情報が保存されます。
** ブロック ドライブには、アプリケーション ボリュームのデータ ブロックが保存されます。
